# 정보구조 개편 기록

처음에는 문서형 포트폴리오 구조로 사이트를 만들었지만, 현재 목적은 인터넷의 여러 정보를 수집해 나와 다른 사용자에게 제공하는 쪽에 더 가깝습니다.

## 왜 바꿨나

- 프로젝트 소개보다 관심사 큐레이션 비중이 커졌습니다.
- 사용자는 "어떤 프로젝트를 했는가"보다 "어떤 주제를 다루는가"를 먼저 보게 됩니다.
- 그래서 `News / Topics / Hubs / Blog` 구조가 더 자연스러워졌습니다.

## 이번 변경의 핵심

- 메뉴를 관심사 중심으로 재구성
- 홈에 최신 업데이트 카드 추가
- 폴더와 Markdown 기반 콘텐츠 운영 구조 정리

## 다음 단계

이제 구조 변경은 끝났고, 각 섹션에 실제 콘텐츠를 채우는 작업이 핵심입니다.
