# 사이트 운영 원칙

## 목적

이 사이트는 GitHub 프로젝트를 단순 나열하지 않고, 각 프로젝트가 왜 존재하는지 설명하는 문서형 포트폴리오를 유지하는 것을 목표로 합니다.

## 운영 기준

- 프로젝트 추가 시 `projects/index.md`와 개별 상세 문서를 함께 갱신합니다.
- 상세 문서는 공통 frontmatter와 공통 섹션 구조를 유지합니다.
- 프로젝트 설명은 기능 나열보다 문제 정의와 구현 의사결정을 우선합니다.
- 운영 메모, 배포 절차, 작성 규칙은 `archive/` 아래에 유지합니다.

## 업데이트 흐름

1. 프로젝트 문서 작성 또는 수정
2. 로컬에서 `npm run docs:build` 확인
3. GitHub에 push
4. Cloudflare Pages 자동 배포 확인
