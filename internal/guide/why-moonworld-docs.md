# 왜 Moonworld Docs인가

GitHub 저장소 목록만으로는 프로젝트의 목적과 구현 의도가 충분히 드러나지 않습니다. Moonworld Docs는 그 빈칸을 문서로 채우기 위해 만들었습니다.

## 핵심 원칙

- 저장소보다 설명을 우선합니다.
- 기능 목록보다 문제 정의와 판단 근거를 남깁니다.
- 배포와 운영 경험도 프로젝트 문서의 일부로 취급합니다.

## 이 구조를 택한 이유

일반적인 포트폴리오 사이트는 보기에는 좋지만, 구현 맥락을 압축하는 데 그치는 경우가 많습니다. 반대로 문서 사이트는 설명력은 좋지만 첫 진입이 딱딱해지기 쉽습니다. 그래서 공개 사이트는 주제 중심으로, 내부 문서는 구조 설명 중심으로 분리했습니다.
